## How to write unit tests

When adding or modifying code, also add or update the related unit tests:

- Focus tests on typical use cases. Then, briefly test realistic corner cases.
- Add validations and assertions to production code where necessary to catch incorrect input.
- Repair and expand deficient tests for the code you edit.
- If the code is hard to test, rewrite it to make it testable.
- Skip testing code that cannot be made testable or requires extensive setup and mocking.
